Iako WebForge moze raditi direktno na Windowsu, preporucujemo [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) za najbolje iskustvo. WSL daje Linux okruzenje koje glatko radi sa WebForge funkcijama.

:::tip[Za≈°to WSL?]
WSL nudi bolje performanse fajl sistema, punu terminalsku podrsku i kompatibilnost s razvojnim alatima na koje se WebForge oslanja.
:::

---

## Postavljanje

<Steps>

1. **Instalirajte WSL**

   Ako vec niste, [instalirajte WSL](https://learn.microsoft.com/en-us/windows/wsl/install) prema zvanicnom Microsoft vodicu.

2. **Instalirajte WebForge u WSL-u**

   Kad je WSL spreman, otvorite WSL terminal i instalirajte WebForge jednom od [metoda instalacije](/docs/).

   ```bash
   curl -fsSL https://opencode.ai/install | bash
   ```

3. **Koristite WebForge iz WSL-a**

   Idite u direktorij projekta (Windows fajlovima pristupate preko `/mnt/c/`, `/mnt/d/` itd.) i pokrenite WebForge.

   ```bash
   cd /mnt/c/Users/YourName/project
   webforge
   ```

</Steps>

## Desktop aplikacija + WSL Server

Ako preferirate WebForge Desktop aplikaciju, ali zelite da server radi u WSL-u:

1. **Pokrenite server u WSL-u** sa `--hostname 0.0.0.0` da dozvolite vanjske konekcije:

   ```bash
   webforge serve --hostname 0.0.0.0 --port 4096
   ```

2. **Povezite Desktop aplikaciju** na `http://localhost:4096`

:::note
Ako `localhost` ne radi u vasem setupu, povezte se preko WSL IP adrese (u WSL-u: `hostname -I`) i koristite `http://<wsl-ip>:4096`.
:::

:::caution
Kada koristite `--hostname 0.0.0.0`, postavite `WEBFORGE_SERVER_PASSWORD` da zastitite server.

```bash
WEBFORGE_SERVER_PASSWORD=your-password webforge serve --hostname 0.0.0.0
```

:::

## Web klijent + WSL

Za najbolje web iskustvo na Windowsu:

1. **Pokrenite `webforge web` u WSL terminalu** umjesto u PowerShell-u:

   ```bash
   webforge web --hostname 0.0.0.0
   ```

2. **Otvorite iz Windows browsera** na `http://localhost:<port>` (WebForge ispisuje URL)

Pokretanje `webforge web` iz WSL-a osigurava ispravan pristup fajl sistemu i terminalsku integraciju, a i dalje je dostupno iz Windows browsera.

## Pristup Windows fajlovima

WSL moze pristupiti svim Windows fajlovima kroz `/mnt/` direktorij:

- Disk `C:` ‚Üí `/mnt/c/`
- Disk `D:` ‚Üí `/mnt/d/`
- I tako dalje...

Primjer:

```bash
cd /mnt/c/Users/YourName/Documents/project
webforge
```

:::tip
Za najgladje iskustvo, razmislite da klonirate/kopirate repo u WSL fajl sistem (npr. pod `~/code/`) i tu pokrenete WebForge.
:::

## Savjeti

- Drzite WebForge u WSL-u za projekte na Windows diskovima - pristup fajlovima je jednostavan
- Koristite VS Code [WSL ekstenziju](https://code.visualstudio.com/docs/remote/wsl) uz WebForge za integrisan tok rada
- Vase WebForge konfiguracije i sesije cuvaju se u WSL okruzenju na `~/.local/share/webforge/`
